# 最简抓取流程

这份说明只保留你们三个人三周内最容易跑通的部分。

## 一、先做什么

不要一开始抓全国。

先选：

- `2` 个城市
- `2` 个岗位关键词
- `1` 个平台

建议第一轮只试：

- 城市：北京、杭州
- 关键词：`Python`、`运营`

## 二、你们真正要做的事情

### 第一步：手工准备搜索页链接

在浏览器里打开目标平台的搜索结果页，把链接复制到 `data/input/seeds_example.csv`。

每一行至少保留：

- `platform`
- `page_type`
- `url`
- `city`
- `keyword`

`page_type` 在第一步统一填 `list`。

### 第二步：先抓列表页

运行：

```bash
python3 src/fetch_pages.py --seed-file data/input/seeds_example.csv --manifest data/raw/list_fetch_manifest.jsonl
```

抓到的 HTML 会放到 `data/raw/html/` 下面。

### 第三步：自己看网页结构，填写选择器

打开浏览器开发者工具，找到：

- 每条职位卡片的外层节点
- 职位详情页链接所在节点

然后修改 `data/input/platform_template.json` 里的：

- `list_page.item_selector`
- `list_page.detail_link_selector`
- `list_page.detail_link_attr`
- `list_page.base_url`

### 第四步：提取详情页链接

运行：

```bash
python3 src/extract_links.py --manifest data/raw/list_fetch_manifest.jsonl --config data/input/platform_template.json --output data/raw/links/detail_urls.csv
```

### 第五步：抓详情页

运行：

```bash
python3 src/fetch_pages.py --seed-file data/raw/links/detail_urls.csv --manifest data/raw/detail_fetch_manifest.jsonl
```

### 第六步：继续填详情页选择器

在浏览器里打开一个职位详情页，找到下面这些字段对应的 CSS 选择器：

- 职位名
- 公司名
- 城市
- 薪资
- 学历
- 经验
- 公司行业
- 公司规模
- 标签
- 岗位描述

然后修改 `platform_template.json` 的 `detail_page` 部分。

### 第七步：解析字段

运行：

```bash
python3 src/parse_details.py --manifest data/raw/detail_fetch_manifest.jsonl --config data/input/platform_template.json --output data/raw/records/jobs_raw.jsonl
```

### 第八步：清洗和去重

运行：

```bash
python3 src/clean_jobs.py --input data/raw/records/jobs_raw.jsonl --output data/processed/jobs_clean.csv
```

## 三、你们现在最该注意的事

- 先追求能跑通，不追求一次抓很多。
- 第一轮只要抓到 `300` 到 `1000` 条有效 JD，就已经足够验证流程。
- 任何一步卡住，都不要继续往后堆任务。
- 抓取时尽量慢一点，别把请求频率开太高。

## 四、每个人分工

### 人1

负责找页面、复制种子链接、填选择器。

### 人2

负责跑脚本、看报错、保存结果。

### 人3

负责检查清洗结果是否靠谱，顺手开始整理技能词表。

## 五、如果你们完全不会找选择器

用浏览器右键元素，点击“检查”。

优先找：

- 有类名的外层块
- 职位标题节点
- 公司名节点
- 薪资节点
- 正文描述块

如果类名像随机字符串，经常会变，就尽量找更稳定的父节点或包含文本结构的区域。

## 六、你们现在不用做的事

- 不用立刻接大模型抽取
- 不用立刻做全国城市面板
- 不用立刻补 CLDS
- 不用一开始就追求自动翻页和全站抓取

先把最基础的 `jobs_clean.csv` 做出来。
