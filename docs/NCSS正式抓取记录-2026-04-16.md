# NCSS 正式抓取记录（2026-04-16）

## 1. 这次抓了什么

为了在三周内尽快拿到一批可用的招聘文本，本项目先选用 `国家大学生就业服务平台（NCSS）` 作为第一主线数据源。

原因很直接：

- 页面和接口结构相对稳定
- 详情页可以直接抓到公开 HTML
- 能较快覆盖多城市、多岗位关键词
- 比起同时做多个平台，更适合当前竞赛时间

本次正式抓取的范围是：

- 城市：`北京`、`上海`、`广州`、`深圳`、`武汉`、`成都`
- 关键词：`Python`、`数据分析`、`运营`、`行政`、`会计`、`产品经理`
- 每个“城市 × 关键词”组合最多抓取 `3` 页列表
- 列表接口每页 `20` 条

批量配置文件见：`data/input/ncss_batch_config.json`

## 2. 已安装依赖

当前只需要一个最小依赖：

```bash
python3 -m pip install -r requirements.txt
```

`requirements.txt` 当前内容为：

- `beautifulsoup4>=4.12`

## 3. 实际使用的数据接口

NCSS 列表页入口：

- `https://www.ncss.cn/student/jobs/index.html`

本次实际调用的列表接口：

- `https://www.ncss.cn/student/jobs/jobslist/ajax/`

接口参数由 `data/input/ncss_batch_config.json` 控制，其中本次关键设置是：

- `areaCode`：城市代码
- `jobName`：岗位关键词
- `offset`：页码
- `limit=20`
- `sourcesName=""`

说明：

- 实测 `sourcesName=""` 的返回更稳定
- 如果把 `sourcesName` 写成 `0`，部分查询会明显变少

## 4. 本次正式执行命令

### 4.1 抓取列表并导出详情页种子

```bash
python3 src/fetch_ncss_jobs.py \
  --config data/input/ncss_batch_config.json \
  --list-output data/raw/records/ncss_listings_raw.jsonl \
  --detail-seed-output data/input/ncss_detail_urls_generated.csv \
  --summary-output data/processed/ncss_list_query_summary.csv \
  --max-jobs 600
```

执行结果：

- 列表原始记录：`2080` 条
- 去重后唯一职位：`2074` 条
- 初始详情页种子：`600` 条

### 4.2 重新做均衡抽样

初始 `600` 条详情页种子按写入顺序截断，会偏向前几个城市，因此又补做了一次均衡抽样。

```bash
python3 src/build_balanced_ncss_detail_seeds.py \
  --input data/raw/records/ncss_listings_raw.jsonl \
  --output data/input/ncss_detail_urls_balanced.csv \
  --max-jobs 600
```

均衡种子结果：

- 总数：`600`
- 城市分布：北京 `102`、上海 `102`、广州 `102`、成都 `102`、武汉 `96`、深圳 `96`
- 关键词分布：`Python`、`产品经理`、`会计`、`数据分析`、`行政`、`运营` 各 `100`

### 4.3 抓取详情页 HTML

```bash
python3 src/fetch_pages.py \
  --seed-file data/input/ncss_detail_urls_balanced.csv \
  --manifest data/raw/ncss_detail_manifest_balanced.jsonl \
  --config data/input/platform_ncss_detail.json \
  --delay 0.2 \
  --skip-existing
```

说明：

- 使用了 `--skip-existing`
- 因此 manifest 里有一部分记录是“直接复用此前已下载 HTML”，不是重新联网抓取

结果口径：

- manifest 记录数：`600`
- 本地 HTML 存在数：`600`
- 本次新抓取：`430`
- 复用已有文件：`170`
- 缺失文件：`0`

### 4.4 解析详情页

```bash
python3 src/parse_details.py \
  --manifest data/raw/ncss_detail_manifest_balanced.jsonl \
  --config data/input/platform_ncss_detail.json \
  --output data/raw/records/ncss_jobs_balanced_raw.jsonl
```

执行结果：

- 成功解析详情页：`600`

### 4.5 清洗和标准化

```bash

  --output data/processed/ncss_jobs_balanced_clean.csv
```

执行结果：

## 8. 当前限制

- 目前主线只跑通了 `NCSS`
- `中国公共招聘网` 在当前环境下出现过 `SSL EOF`，暂时不适合做第一主线
- 这批样本是“竞赛冲刺版样本”，目标是先快速形成可分析数据，不是做全国全量抓取
- `city_std` 有一部分是根据页面文本和种子城市共同标准化出来的，后续还可以继续细化
- 当前关键词仍然偏向通用岗位，后续可再补 `新媒体`、`测试`、`销售`、`客服`、`人力` 等岗位群
