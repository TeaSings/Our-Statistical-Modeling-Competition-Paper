# 中文数据来源与脚本说明

这份文档回答两个问题：

1. 现阶段项目优先抓哪些中文网站
2. 仓库里现有脚本分别怎么接这些网站

## 1. 当前最推荐的数据源

### NCSS

- 网址：<https://www.ncss.cn/student/jobs/index.html>
- 角色：当前主线招聘数据源
- 优点：公开、结构稳定、详情页正文质量较高
- 当前仓库状态：已经跑通列表抓取、详情抓取、字段解析、清洗

### 中国公共招聘网

- 网址：<https://job.mohrss.gov.cn/cjobs/jobinfolist/listJobinfolist>
- 角色：后续可补充的官方招聘源
- 当前仓库状态：保留了样例种子和参考页面，不作为当前主线

### 智联公开详情页

- 入口：<https://www.zhaopin.com/jobs>
- 角色：市场化岗位的少量补样本来源
- 当前仓库状态：保留了样例详情页，不建议现阶段做全站爬虫

### 国家统计局与普查数据

- 国家数据：<https://data.stats.gov.cn/>
- 普查入口：<https://www.stats.gov.cn/sj/pcsj/>
- 角色：补城市控制变量、行业分类和背景数据
- 当前仓库状态：已保存参考页面快照

## 3. NCSS 主线脚本怎么用

### 第一步：抓列表接口

```bash
python3 src/fetch_ncss_jobs.py \
  --config data/input/ncss/ncss_batch_config.json
```

如果要做全地区扫描，用：

```bash
python3 src/fetch_ncss_jobs.py \
  --config data/input/ncss/ncss_batch_config_all_areas.json \
  --list-output data/raw/ncss/records/ncss_listings_all_areas_raw.jsonl \
  --detail-seed-output data/input/ncss/ncss_detail_urls_all_areas.csv \
  --summary-output data/processed/ncss/ncss_list_query_summary_all_areas.csv \
  --max-jobs 0
```

### 第二步：抓详情页 HTML

```bash
python3 src/fetch_pages.py \
  --seed-file data/input/ncss/ncss_detail_urls_balanced.csv \
  --manifest data/raw/ncss/manifests/ncss_detail_manifest_balanced.jsonl \
  --output-dir data/raw/ncss/html \
  --config data/input/ncss/platform_ncss_detail.json \
  --delay 0.2 \
  --skip-existing
```

### 第三步：解析详情页

```bash
python3 src/parse_details.py \
  --manifest data/raw/ncss/manifests/ncss_detail_manifest_balanced.jsonl \
  --config data/input/ncss/platform_ncss_detail.json \
  --output data/raw/ncss/records/ncss_jobs_balanced_raw.jsonl \
  --overwrite
```

`parse_details.py` 现在支持一次传多个 manifest，适合把全地区主任务和分片补抓一起解析。

### 第四步：清洗成分析表

```bash
python3 src/clean_jobs.py \
  --input data/raw/ncss/records/ncss_jobs_balanced_raw.jsonl \
  --output data/processed/ncss/ncss_jobs_balanced_clean.csv
```

## 4. 其他脚本的作用

- `src/extract_ncss_area_codes.py`：从 NCSS 列表页 HTML 提取全部地区码
- `src/build_balanced_ncss_detail_seeds.py`：把详情页种子做均衡抽样
- `src/build_manual_seed_sheet.py`：生成手工补链任务表
- `src/parse_job_pages_by_text.py`：对非 NCSS 页面做文本规则解析

## 5. 当前最务实的用法

- 现在就要分析，直接从 `data/processed/ncss/ncss_jobs_balanced_clean.csv` 开始
- 需要说明覆盖范围，配合 `data/processed/ncss/ncss_list_query_summary_all_areas.csv`
- 需要继续榨干 NCSS，使用 `data/input/ncss/shards/` 里的详情分片种子
