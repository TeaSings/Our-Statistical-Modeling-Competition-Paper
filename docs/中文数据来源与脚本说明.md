# 中文数据来源与脚本说明

这份文档专门回答两个问题：

1. 现阶段在中文互联网上，哪些数据网站最值得抓。
2. 仓库里现在已经给你们补了哪些脚本，可以怎么用。

## 一、先说结论

如果你们想仿照参考论文的思路，但时间又只有三周，最稳的数据路线是：

- 用 `国家职业分类大典` 和 `国民经济行业分类` 做标签标准。
- 用 `中国公共招聘网` 和 `国家大学生就业服务平台` 抓公开招聘文本。
- 用 `智联招聘公开职位详情页` 补充市场化岗位样本。
- 用 `国家数据` 和 `普查数据` 补城市层面的控制变量。

这套组合比直接死磕商业平台全站爬虫更现实，也更适合三位大二同学在短时间内跑通。

## 二、最推荐的数据网站

### 1. 中国公共招聘网

- 网址：<https://job.mohrss.gov.cn/cjobs/jobinfolist/listJobinfolist>
- 类型：官方招聘岗位库
- 适合做什么：抓岗位名称、月薪范围、地区、单位类型、学历要求、行业类别、岗位详情文本
- 优点：公开、筛选项多、官方平台
- 难点：列表页有 `iframe`，不适合一开始就做复杂自动化

同时它还有一个很有用的页面：

- 招聘排行榜：<https://job.mohrss.gov.cn/cjobs/institution/rankingLists>

这个页面直接给出区域、行业、职业层面的起薪、招聘人数、热门岗位排名，适合写背景分析和补宏观图表。

### 2. 国家大学生就业服务平台

- 网址：<https://www.ncss.cn/student/jobs/index.html>
- 类型：高校毕业生岗位平台
- 适合做什么：抓校招、应届、实习、全职岗位文本
- 优点：文本质量通常较高，岗位描述比较完整
- 难点：城市字段有时不如商业平台完整，需要保留 `city_seed`

### 3. 智联招聘公开职位详情页

- 入口：<https://www.zhaopin.com/jobs>
- 类型：市场化招聘平台
- 适合做什么：补充若干公开职位详情页样本
- 优点：搜索引擎可以搜到部分详情页，文本结构清晰
- 难点：不建议你们三周内去做智联全站爬虫，只适合手工补一批详情页

### 4. 国家数据

- 网址：<https://data.stats.gov.cn/>
- 类型：官方统计数据门户
- 适合做什么：补人口、GDP、工资、产业结构等城市控制变量

### 5. 国家统计局普查数据

- 网址：<https://www.stats.gov.cn/sj/pcsj/>
- 类型：官方普查数据入口
- 适合做什么：补城市人口、教育结构、年龄结构变量

### 6. 职业分类与行业分类页面

- 职业分类大典背景页：<https://www.ndrc.gov.cn/fggz/jyysr/jysrsbxf/202210/t20221031_1340470.html>
- 行业分类页面：<https://www.stats.gov.cn/xxgk/tjbz/gjtjbz/201710/t20171017_1758922.html>

它们虽然不是招聘数据本身，但对职业映射和行业映射非常关键。

## 二点五、`2026年4月16日` 的实际抓取验证

我今天用仓库里的脚本对一小批公开页面做了联网验证，结果如下：

- `NCSS` 公开职位详情页：当前可抓，返回 `HTTP 200`
- 国家统计局普查数据页：当前可抓，返回 `HTTP 200`
- 国家统计局行业分类页：当前可抓，返回 `HTTP 200`
- 中国公共招聘网详情页：当前环境下出现 `SSL EOF`，说明它依然值得保留，但不建议当作你们第一条唯一主线

所以如果你们今天就要开始实抓，我建议优先级是：

1. `NCSS`
2. `国家统计局官方页面`
3. `中国公共招聘网`
4. `智联公开详情页`

## 三、仓库里新加了什么

### 1. 数据源注册表

文件：

- `data/input/job_source_registry.json`

用途：

- 记录每个可用网站的入口网址、优先级、用途和注意事项。

### 2. 官方参考页面种子

文件：

- `data/input/reference_pages.csv`

用途：

- 用现有的 `src/fetch_pages.py` 一次性下载官方页面留档。

运行：

```bash
python3 src/fetch_pages.py --seed-file data/input/reference_pages.csv --manifest data/raw/manual_reference_manifest.jsonl
```

### 3. 人工搜集种子任务表生成脚本

脚本：

- `src/build_manual_seed_sheet.py`

用途：

- 根据城市、关键词、平台，生成一张“谁去打开哪个网站、搜什么词、最后把真实结果链接贴回来”的任务表。

运行：

```bash
python3 src/build_manual_seed_sheet.py --cities 北京,上海,杭州 --keywords Python,数据分析,运营
```

默认会输出：

- `data/input/manual_seed_tasks.csv`

### 4. 详情页示例链接

文件：

- `data/input/detail_urls_example.csv`

用途：

- 这里放了几条当前能公开访问的职位详情页示例，方便你们先测试流程，不用先自己找链接。

### 5. 基于页面可见文本的详情页解析脚本

脚本：

- `src/parse_job_pages_by_text.py`

支持的平台：

- `mohrss_public`
- `ncss_jobs`
- `zhaopin_detail`

适合的场景：

- 你们已经抓到了详情页 HTML，但还没来得及研究 CSS 选择器。
- 页面结构相对稳定，能从可见文本里直接提取关键信息。

运行：

```bash
python3 src/fetch_pages.py --seed-file data/input/detail_urls_example.csv --manifest data/raw/manual_detail_manifest.jsonl
python3 src/parse_job_pages_by_text.py --manifest data/raw/manual_detail_manifest.jsonl --output data/raw/records/manual_jobs_text_raw.jsonl
python3 src/clean_jobs.py --input data/raw/records/manual_jobs_text_raw.jsonl --output data/processed/manual_jobs_text_clean.csv
```

### 6. NCSS 详情页选择器配置

文件：

- `data/input/platform_ncss_detail.json`

用途：

- 这是基于当前抓到的 `NCSS` 详情页结构整理出来的选择器配置。
- 如果你们手上已经有一批 `NCSS` 详情页链接，可以直接抓详情页，再用 `parse_details.py` 做选择器解析。

运行：

```bash
python3 src/fetch_pages.py --seed-file data/input/detail_urls_example.csv --manifest data/raw/example_ncss_detail_manifest.jsonl --config data/input/platform_ncss_detail.json
python3 src/parse_details.py --manifest data/raw/example_ncss_detail_manifest.jsonl --config data/input/platform_ncss_detail.json --output data/raw/records/example_ncss_jobs_raw.jsonl
python3 src/clean_jobs.py --input data/raw/records/example_ncss_jobs_raw.jsonl --output data/processed/example_ncss_jobs_clean.csv
```

说明：

- 这份配置主要适用于 `NCSS` 详情页，不是列表页全自动抓取配置。
- 也就是说，它最适合“你们已经搜集到若干详情页链接”的场景。

## 四、建议你们怎么实际开工

### 第一步：先抓官方页面和示例详情页

这样做的目的不是立刻形成大样本，而是先验证脚本通不通。

建议先运行：

```bash
python3 src/fetch_pages.py --seed-file data/input/reference_pages.csv --manifest data/raw/manual_reference_manifest.jsonl
python3 src/fetch_pages.py --seed-file data/input/detail_urls_example.csv --manifest data/raw/manual_detail_manifest.jsonl
python3 src/parse_job_pages_by_text.py --manifest data/raw/manual_detail_manifest.jsonl --output data/raw/records/manual_jobs_text_raw.jsonl
python3 src/clean_jobs.py --input data/raw/records/manual_jobs_text_raw.jsonl --output data/processed/manual_jobs_text_clean.csv
```

### 第二步：再扩展到人工搜集真实搜索页

你们先生成任务表：

```bash
python3 src/build_manual_seed_sheet.py --cities 北京,上海,广州,深圳,杭州 --keywords Python,数据分析,运营,行政,会计
```

然后三个人分工：

- 一个人负责中国公共招聘网
- 一个人负责国家大学生就业服务平台
- 一个人负责手工补智联公开详情页

每个人把真实搜索结果页或详情页链接补进种子表，再用现有抓取脚本抓取。

## 五、关于 CLDS 和商业平台的现实建议

### CLDS

- 可以了解，不建议三周内作为主数据依赖。
- 真要用，也更适合做论文讨论里的补充说明。

### 商业平台全站爬虫

- 不建议你们当前阶段直接做。
- 你们更应该先吃透 `官方公开平台 + 少量公开市场化详情页` 这条线。
- 先做出一份能用于技能抽取的 `jobs_clean.csv`，比“理论上能抓更多平台”更重要。
