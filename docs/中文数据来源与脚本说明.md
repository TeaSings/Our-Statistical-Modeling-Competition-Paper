# 中文数据来源与脚本说明

这份文档回答两个问题：

1. 现阶段项目优先抓哪些中文网站
2. 仓库里现有脚本分别怎么接这些网站

## 1. 当前最推荐的数据源

### NCSS

- 网址：<https://www.ncss.cn/student/jobs/index.html>
- 角色：当前主线招聘数据源
- 优点：公开、结构稳定、详情页正文质量较高
- 当前仓库状态：已经跑通列表抓取、详情抓取、字段解析、清洗

### 51job

- 网址：<https://we.51job.com/>
- 角色：补充市场化招聘岗位，尤其适合获取更贴近社会招聘的岗位正文
- 特点：页面质量高，但存在 `405`、滑块验证和会话限制
- 当前仓库状态：校招专题页样本已稳定；社招方案采用浏览器附着 + 顺序低速调度，支持断点续跑和 watcher 进度条

### 中国公共招聘网

- 网址：<https://job.mohrss.gov.cn/cjobs/jobinfolist/listJobinfolist>
- 角色：后续可补充的官方招聘源
- 当前仓库状态：保留了样例种子和参考页面，不作为当前主线

### 智联公开详情页

- 入口：<https://www.zhaopin.com/jobs>
- 角色：市场化岗位的少量补样本来源
- 当前仓库状态：保留了样例详情页，不建议现阶段做全站爬虫

### 国家统计局与普查数据

- 国家数据：<https://data.stats.gov.cn/>
- 普查入口：<https://www.stats.gov.cn/sj/pcsj/>
- 角色：补城市控制变量、行业分类和背景数据
- 当前仓库状态：已保存参考页面快照

## 3. NCSS 主线脚本怎么用

### 第一步：抓列表接口

```bash
python3 src/platforms/ncss/fetch_ncss_jobs.py \
  --config data/input/ncss/ncss_batch_config.json
```

如果要做全地区扫描，用：

```bash
python3 src/platforms/ncss/fetch_ncss_jobs.py \
  --config data/input/ncss/ncss_batch_config_all_areas.json \
  --list-output data/raw/ncss/records/ncss_listings_all_areas_raw.jsonl \
  --detail-seed-output data/input/ncss/ncss_detail_urls_all_areas.csv \
  --summary-output data/processed/ncss/ncss_list_query_summary_all_areas.csv \
  --max-jobs 0
```

### 第二步：抓详情页 HTML

```bash
python src/platforms/ncss/fetch_pages.py \
  --seed-file data/input/ncss/ncss_detail_urls_all_areas.csv \
  --manifest data/raw/ncss/manifests/ncss_detail_manifest_all_areas.jsonl \
  --output-dir data/raw/ncss/html \
  --config data/input/ncss/platform_ncss_detail.json \
  --skip-existing \
  --workers 16
```

### 第三步：解析详情页

```bash
python src/platforms/ncss/parse_details.py \
  --seed-file data/input/ncss/ncss_detail_urls_all_areas.csv \
  --config data/input/ncss/platform_ncss_detail.json \
  --output data/raw/ncss/records/ncss_jobs_all_areas_raw.jsonl \
  --overwrite \
  --workers 16
```

`parse_details.py` 现在既支持一次传多个 manifest，也支持直接从主种子和本地 HTML 解析，适合在 manifest 跨机器失效后继续恢复。

### 第四步：清洗成分析表

```bash
python src/clean_jobs.py \
  --input data/raw/ncss/records/ncss_jobs_all_areas_raw.jsonl \
  --output data/processed/ncss/ncss_jobs_all_areas_clean.csv
```

## 4. 其他脚本的作用

- `src/platforms/ncss/extract_ncss_area_codes.py`：从 NCSS 列表页 HTML 提取全部地区码
- `src/platforms/ncss/build_balanced_ncss_detail_seeds.py`：把详情页种子做均衡抽样
- `src/tools/build_manual_seed_sheet.py`：生成手工补链任务表
- `src/tools/parse_job_pages_by_text.py`：对非 NCSS 页面做文本规则解析

## 5. 51job 当前怎么用

如果要跑 51job 社招，优先使用顺序调度器而不是高并发 requests：

```powershell
python src/platforms/job51/run_sequential_social_crawl.py `
  --transport browser `
  --browser-cdp-url http://127.0.0.1:9222 `
  --browser-min-interval 0.6 `
  --browser-max-retries 4 `
  --manual-verify `
  --manual-verify-wait 120 `
  --workers 1 `
  --page-size 50 `
  --specific-only
```

实时进度查看：

```powershell
python src/platforms/job51/watch_51job_progress.py --mode social --interval 5
```

只看当前快照：

```powershell
python src/platforms/job51/watch_51job_progress.py --mode social --once
```

## 6. 当前最务实的用法

- 现在就要做正文分析，直接从 `data/processed/ncss/ncss_jobs_all_areas_clean.csv` 开始
- 需要说明覆盖范围，配合 `data/processed/ncss/ncss_listings_all_areas_flat.csv` 和 `data/processed/ncss/ncss_list_query_summary_all_areas.csv`
- 需要继续重建或核验原始层，优先使用 `data/input/ncss/ncss_detail_urls_all_areas.csv`、`src/platforms/ncss/rebuild_local_manifest.py` 和 `src/platforms/ncss/watch_ncss_progress.py`
- 需要继续积累市场化岗位样本时，再开启 `51job` 的浏览器顺序抓取链路
