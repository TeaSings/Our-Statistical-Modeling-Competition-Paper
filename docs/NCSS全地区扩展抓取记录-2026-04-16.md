# NCSS 全地区扩展抓取记录（2026-04-16）

这份记录对应本项目对 `NCSS` 的第二阶段扩展工作，目标是把原来只覆盖少数一线城市的样本，扩大到尽可能完整的全地区列表扫描，并同步重构 `data/` 目录结构。

## 1. 扩展思路

原来的局限主要有两点：

- 城市只覆盖少量超一线和一线城市
- 每个城市和关键词组合只抓少数页面

为了解决这个问题，本次扩展采用了两步：

1. 从 NCSS 列表页 HTML 中提取全部公开地区码
2. 对全部地区码执行空关键词列表扫描，再导出全量详情页种子

## 2. 关键输入文件

- `data/input/ncss/ncss_area_codes_all.json`
- `data/input/ncss/ncss_area_codes_all.csv`
- `data/input/ncss/ncss_batch_config_all_areas.json`
- `data/input/ncss/ncss_detail_urls_all_areas.csv`

其中：

- `ncss_area_codes_all.*` 来自 `src/platforms/ncss/extract_ncss_area_codes.py`
- `ncss_batch_config_all_areas.json` 指向全地区地区码文件，并设置空关键词扫描

- 一条主任务：顺序抓取全地区详情页
- 多条分片任务：把剩余种子拆到 `data/input/ncss/shards/` 下并发补抓

对应 manifest 目录：

- `data/raw/ncss/manifests/ncss_detail_manifest_all_areas.jsonl`
- `data/raw/ncss/manifests/shards/`
