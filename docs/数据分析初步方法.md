**3. 方法归纳**

| 方法 | 解决什么问题 | 数据条件 | 优点 | 局限 | 竞赛适配 |
|---|---|---|---|---|---|
| 描述统计与结构测度 | 看技能、学历、经验、薪资如何变化 | 有标准化岗位数据即可 | 最稳、最直观 | 解释性强但因果弱 | 很高 |
| 招聘文本词典提取 | 识别技能与任务 | 有岗位描述文本 | 成本低、可控 | 召回率有限 | 很高 |
| 大模型信息抽取 | 抽取技能、任务、AI 使用场景、岗位类型 | 有文本，且能做人工抽查 | 灵活、效果好 | 会幻觉，需校验 | 很高 |
| 暴露度构建 | 识别哪些岗位更受 GenAI 冲击 | 岗位标准化后可映射 | 适合做 DID | 暴露不是实际采用 | 高 |
| 面板固定效应 | 比较城市/岗位组随时间变化 | 城市-季度或城市-岗位-季度面板 | 稳健、易解释 | 因果性有限 | 很高 |
| DID | 识别 GenAI 冲击前后差异 | 有前后期、有高低暴露组 | 竞赛里最容易出层次 | 平行趋势要求高 | 很高 |
| 事件研究 | 看冲击动态路径 | 时间维度更细 | 图形效果好 | 样本要求更高 | 高 |
| PCA/熵值法/综合指数 | 构建“技能重塑指数” | 多维指标 | 论文展示效果好 | 指数解释需谨慎 | 很高 |
| 异质性/机制分析 | 看哪些城市、岗位、行业受影响更大 | 样本足够细 | 提升论文深度 | 容易碎片化 | 很高 |

**4. 文献结论对你的题目的直接指向**

基于本轮检索，我的判断是：
- 最成熟的研究主线不是“AI 是否让就业总量下降”，而是“AI 如何重塑技能需求结构、任务边界和岗位分层”。
- 最适合你这个题目的数据路径不是宏观就业总量，而是“在线招聘文本 + 城市面板 + 暴露度指标”。
- 最适合本科生团队的识别方式不是做特别重的结构模型，而是“`技能抽取 + 指数构建 + 双向固定效应/暴露度 DID + 异质性`”。

**二、本题研究可行性评估**

| 维度 | 评估 | 说明 |
|---|---|---|
| 数据可得性 | 中上 | 公开招聘页可得，但 `2021-2023` 历史页面相对难；`2024-2026` 更容易。且截至 `2026-04-16`，`2026-04-17` 到 `2026-12-31` 的数据还无法实际获取，只能后续滚动补齐。 |
| 变量可构造性 | 高 | 岗位名称、JD、技能、学历、经验、薪资、城市、企业、日期都能标准化；AI 相关技能和任务也能提取。 |
| 方法可操作性 | 高 | 文本挖掘、LLM 抽取、描述统计、PCA/熵值法、面板 FE、DID 都适合本科团队。 |
| 创新性 | 高 | “生成式 AI + 招聘文本 + 城市技能结构 + LLM 抽取”组合有明显新意。 |
| 国家战略贴合度 | 很高 | 对接“人工智能+”“就业优先”“数字中国”“人才强国”。 |
| 政策建议空间 | 很高 | 可落到城市人才政策、职业培训、教育改革、公共就业服务。 |
| 竞赛可落地性 | 很高 | 只要控制样本范围，不追求全量抓取，非常适合统计建模大赛。 |

结论：
- 这个题目是“可做、能做出层次、容易出政策建议”的好题。
- 真正的风险不在方法，而在数据范围失控。
- 最该控制的是：城市数量、平台数量、岗位范围、时间粒度。

**五、模型体系设计与建模步骤**

**主线模型 1：岗位文本标准化与 LLM 信息抽取**

目标：
- 从招聘文本中抽出技能、任务、AI 暴露标签

步骤：
1. 先清洗 `job_title_raw + job_desc_raw + skill_text_raw`
2. 用规则词典抽取基础技能词
3. 用 LLM 进行结构化抽取
4. 人工抽查并修正规则
5. 生成 `job-level structured dataset`

最适合本科生的实现：
- 不训练新模型
- 用规则 + LLM + 人工校验

**主线模型 2：城市技能重塑指数构建**

目标：
- 把分散的岗位技能变化转成城市层面的综合指标

步骤：
1. 按城市-季度聚合技能占比
2. 计算高阶技能、AI 技能、常规技能、技能多样性
3. 标准化各指标
4. 用 PCA 或熵值法合成指数

公式：
```text
SRI_ct = Σm wm * Zm_ct
```

其中：
- `SRI_ct` 为城市 c 在 t 期的技能重塑指数
- `Zm_ct` 为标准化后的第 m 个指标
- `wm` 为权重

**主线模型 3：双向固定效应 / 暴露度 DID**

最推荐的主模型是城市-岗位族-季度面板：

```text
Y_cgt = β1 (Post_t × Exposure_g) + β2 X_ct + μ_c + ν_g + τ_t + ε_cgt
```

含义：
- `Y_cgt`：城市 c、岗位族 g、季度 t 的技能结构指标
- `Post_t`：`2022-11-30` 后
- `Exposure_g`：岗位族 g 的 GenAI 暴露度
- `X_ct`：城市控制变量
- `μ_c`：城市固定效应
- `ν_g`：岗位族固定效应
- `τ_t`：时间固定效应

适合的被解释变量：
- `genai_skill_share`
- `high_order_skill_share`
- `routine_skill_share`
- `salary_mid_month`
- `degree_upgrade`

为什么它最适合：
- 容易做
- 容易解释
- 适合比赛展示
- 不需要特别难拿的企业内部数据

**主线模型 4：城市层 shift-share 冲击模型**

如果你想更“像劳动经济学”，推荐这个：

```text
Shock_ct = Σg share_cg,base × Exposure_g × Post_t
```

再估计：
```text
Y_ct = α + β Shock_ct + γ X_ct + μ_c + τ_t + ε_ct
```

解释：
- 用城市基期岗位结构和岗位暴露度构造“外生冲击强度”
- 很适合解释“为什么不同城市受到的 GenAI 冲击不同”

基期建议：
- 优先用 `2022Q1-Q3`
- 因为 `2022-11-30` 是 ChatGPT 公布时间点
- 如果 `2021` 数据不稳，可以不用强求 2021 做基期

**增强项模型 5：事件研究**

```text
Y_cgt = Σk βk Dk_t × Exposure_g + μ_c + ν_g + τ_t + ε_cgt
```

用途：
- 看冲击是立即发生还是滞后发生
- 检查平行趋势

图形效果很好：
- 是比赛答辩里非常加分的图

**增强项模型 6：异质性与机制模型**

异质性：
- 按城市等级
- 按行业
- 按学历要求
- 按经验要求
- 按企业性质

机制变量：
- `complementary_skill_share`
- `managerial_skill_share`
- `entry_level_share`
- `degree_upgrade`

可写法：
```text
M_cgt = α + β (Post_t × Exposure_g) + FE + ε
```

再看机制变量变化是否解释主效应。

**稳健性检验建议**
- 更换暴露度口径
  - 关键词法
  - LLM 分类法
  - O*NET/Felten 映射法
- 更换时间口径
  - 月度改季度
- 更换样本
  - 去掉 AI 原生岗位
  - 去掉一线城市
- 更换事件时间
  - `2022-11-30`
  - `2023-08-15` 中国《生成式人工智能服务管理暂行办法》实施

**六、适合统计建模大赛的最终研究路线**

最推荐的终版结构：

1. 研究问题
- 生成式 AI 是否正在重塑城市岗位技能结构
- 哪些城市、哪些岗位、哪些技能变化最大
- 这种变化是“替代”还是“互补”

2. 数据路径
- 在线招聘公开数据
- 主样本城市 `30-40`
- 主样本平台 `4-5`
- 时间主线 `2022Q1-2026Q1`

3. 方法路径
- 规则词典 + LLM 技能抽取
- 城市技能重塑指数
- 城市-岗位族-季度 DID/FE
- 异质性与机制分析

4. 论文结构
- 文献与理论机制
- 数据来源与抽取流程
- 描述统计与结构事实
- 指数构建
- 主模型结果
- 异质性与机制
- 稳健性
- 政策建议

主线方法：
- 招聘文本清洗
- LLM 信息抽取
- 指数构建
- 双向固定效应 / DID

增强项：
- 事件研究
- 技能共现网络
- 空间效应
- 合成 DID
- 多平台稳健性

**七、如果数据不足，如何做降级方案**

最稳的降级顺序：

1. 缩城市
- 从全国主要城市降到 `20` 个代表性城市

2. 缩时间
- 从 `2021-2026` 改为 `2022Q1-2026Q1`
- 或直接 `2023-2026Q1`

3. 缩平台
- 只保留 `智联 + 51job + BOSS + 猎聘`

4. 缩岗位
- 只保留 `AI/数据/产品/运营/文案/客服/财务/HR`

5. 缩模型
- 保留“描述统计 + 指数构建 + FE/DID”
- 暂时不做事件研究和空间模型

6. 缩抽取难度
- 先做关键词词典
- 再用 LLM 做补充
- 不做端到端深度学习训练

最低可交付版本：
- `20` 城市
- `4` 平台
- `8` 岗位族
- `2023-2026Q1`
- `LLM 抽取 + 熵值法指数 + 双向固定效应`

这套已经足够形成一篇完整、可答辩、可出政策建议的比赛论文。

**八、如何从结论中提炼政策建议**

建议不要泛泛写“加强培训”。要按实证结果对应政策。

如果结果 1：
- GenAI 高暴露岗位中，高阶认知和沟通技能占比上升
政策建议：
- 城市职业培训从“单纯软件操作”转向“AI 协作 + 表达 + 业务理解”
- 高校课程增加“Prompt、数据素养、任务拆解、跨学科协作”

如果结果 2：
- 入门型白领岗位减少，实习/初级岗位受挤压
政策建议：
- 推出青年岗位过渡计划
- 鼓励企业保留“AI 辅助下的入门岗位”
- 加强实习、见习、学徒制

如果结果 3：
- 一线城市技能升级更快，二三线城市滞后
政策建议：
- 分层推进城市人才政策
- 对二三线城市加大数字技能培训和公共就业服务平台建设

如果结果 4：
- 企业提高学历要求，但真实需求更偏技能
政策建议：
- 推动“以技能证书替代部分学历门槛”
- 建立 AI 时代技能认证体系

如果结果 5：
- AI 原生岗位薪资溢价扩大，普通岗位分化加剧
政策建议：
- 推出针对中间技能劳动者的转岗培训
- 重点扶持制造、物流、客服、财务等高暴露领域的人机协同转型

可以把题目收束为：

`生成式人工智能冲击下城市就业技能结构重塑研究：基于中国主要城市在线招聘文本与大模型信息抽取的证据`

研究主线一句话版：

`基于中国主要城市在线招聘岗位公开数据，运用规则词典与大模型相结合的方法提取岗位技能与任务信息，构建城市就业技能重塑指数和岗位生成式人工智能暴露度指标，采用双向固定效应、暴露度 DID 与异质性分析，识别生成式人工智能对城市岗位技能需求、学历经验门槛和薪资结构的影响机制，并据此提出城市人才培养与就业治理建议。`
